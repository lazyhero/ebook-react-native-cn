React-Native:快用我!

Javascript运行内核

RCTRootView是如何获取到JavaScript代码的呢?从源码得知有两种方式：

  1. 从url获得
  2. 从本地文件获得

默认从url加载,地址为 http://localhost:8081/index.ios.bundle,这哥们哪来的？

答案就潜藏在我们启动应用时一并启动的React packager,忘了?看下面想起来没？ React packager服务的目标就是把应用中的JavaScript代码转换成正确的格式以便让RCTRootView执行,为了达成这个目标,React packager服务要完成几件事

首先它不能仅仅只服务index.ios.js文件中的代码.当应用运行时,打开你的浏览器(chrome终于出马了)访问http://localhost:8081/index.ios.bundle 你会得到一个巨大的数据包(5万多行),如下:

如果你得到的是如下报错信息: 不用担心,在最新的react-native中需要加上平台参数,正确的访问地址为:http://localhost:8081/index.ios.bundle?platform=ios&dev=true

来试着搜搜你在index.ios.js编写的代码

由于编码问题,所以不要搜中文哦~~

除了我们自己编写的代码,其他几万行都是React-Native框架的.

如果React packager服务没有启动,你只要到项目目录下输入npm start即可,命令脚步所在的位置是node_modules/react-native/packager/packager.sh

就像我们所看到的,React packager服务跟随项目构建和启动过程一并启动.有时关闭ios模拟器和xcode，服务并不会自动停止.当你再次启动模拟器时会发现报错,这是因为之前的服务实例仍然占用着端口,应对这种情况,ctrl+c关闭运行服务的终端或者在任务管理器中强制关闭它.

其次它还要帮着转换代码,因为你也发现了,在我们代码里有一些是用ES5标准写的,还有一些是用ES6标准写的……此等混搭风也是醉了……不过这也符合React-Native的尿性——推动ES6标准.

JSX

在index.ios.js中有类似于XML的代码块,如下:

render: function() { return (
    <View style={styles.container}>
      <TextInput style={styles.nameInput} onChange={this.onNameChanged} placeholder='Who should be greeted?'/>
        <Text style={styles.welcome}> Hello, {this.state.name}!</Text> <Text style={styles.instructions}>
        To get started, edit index.ios.js
          </Text>
          <Text style={styles.instructions}>
            Press Cmd+R to reload,{'\n'}
         Cmd+Control+Z for dev menu 
          </Text>
    </View> );
}

这叫JSX,同XML很像.具体可参考官网,JSX没有严格定义语义标签，他也不是给浏览器解析的,更不是ECMAScript标准的一部分,它会被各种预处理器(转换器)处理转换成标准的Javascript代码.

之前提到过React packager会转换源码供RCTRootView使用,实际上这里也会把JSX转换为标准的ES5代码,转换完的代码在http://localhost:8081/index.ios.bundle?platform=ios&dev=true可以看到,就是上图中文编码那.

其中有一些不太易读的部分，但这并不意味着你要花很长时间去适应JSX,以我的体验来讲它使用起来很自然也很便捷，它是React和React-Native框架的核心组成部分,我们会经常用到它的.

ES2015

React packager另一个转换行为是把ES6转换为ES5.ES6提供了模块加载、类、计算属性等等特性

关于最新标准的Javascript有很多种说法，比如ECMAScript 2015,ECMAScript.next,ECMAScript 6, ECMAScript Harmony.

如果你在网上搜会发现ES6被广泛用来描述下一代ECMAScript标准，在本书里我们暂且叫他ES2015.这里提供一个ES2015转ES5的工具

虽然最新版标准还没最终确定,可这阻挡不了Jser们使用它的热情,很多贡献者纷纷提供了ES2015转换ES5的工具,这也就说为什么下面的代码可以工作.

var React = require('react-native'); 
var {
  AppRegistry,
  StyleSheet,
  Text,
  TextInput,
  View,
} = React;

这5个不同变量(括号里的AppRegistry, StyleSheet...)同时被赋值为React.AppRegistry,React.StyleSheet等,这种写法叫做解构赋值,下面是几个简单的例子

var fruits = {banana: "A banana", orange: "An orange", apple: "An apple"};
var { banana, orange, apple } = fruits;

解释下,"A banana"被分配给banana,"An orange"被分配给orange,"An apple"被分配给apple,转换为ES5写法为:

var fruits = { banana: "A banana", orange: "An orange", apple: "An apple" };

var banana = fruits.banana; 
var orange = fruits.orange; 
var apple = fruits.apple;

我们在下面可以直接使用AppRegistry, StyleSheet等等

在index.ios.js中还有一部分代码需要转换,最后一行:

AppRegistry.registerComponent('HelloWorld', () => HelloWorld);

转换为:

AppRegistry.registerComponent('HelloWorld', function() {return HelloWorld;});

这是ES2015提供的另一种很时髦写法——箭头函数,它是一种简化的函数声明方式,但有些许不同

  1. 对 this 的关联。函数内置 this 的值，取决于箭头函数在哪儿定义，而非箭头函数执行的上下文环境。
  2. new 不可用。箭头函数不能使用 new 关键字来实例化对象，不然会报错。
  3. this 不可变。函数内置 this 不可变，在函数体内整个执行环境中为常量。
  4. 没有arguments对象。更不能通过arguments对象访问传入参数。只能使用显式命名或其他ES6新特性来完成。
