React-Native:快用我!

配置开发环境

React-Native除了使用Javascript外，还附加了一些Object-c编写的工具(代码),正是这些代码为你提供创建、运行、调试原生IOS应用的支持。

毫无意外，使用React-Native创建IOS应用也必须配置下开发环境(程序员的梦魇)

在开发机上，需要安装下列组件

  • Homebrew
  • io.js 如果你的node在4.0+ 则已经集成好了 无需额外安装
  • Watchman
  • Chrome 当然使用其他具备调试能力的浏览器也可以
  • React-Native-CLI 命令行工具 后面会详细说明它的作用

下面我们一步步来安装,表急小伙子

Homebrew

这哥们是Mac上公认好用的包管理工具,后面大多数软件都会通过它来安装

打开终端,输入

ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

小伙子看到ruby别着急 mac默认安装ruby

更多关于Homebrew的使用,访问官网自己查

Watchman

看这个名字,大家应该猜到了。React-Native使用Watchman来做热部署(实时反馈源码的改变),细节部分会在后面的章节详述.

brew install --HEAD watchman

HEAD:保证下载的是最新版

React-Native-CLI

同多数工具集提供的命令行工具类似(比如express),React-Native-CLI命令行工具帮助你像这样

react-native init HelloWorld

新建初始化React-Native项目

我们使用npm来安装它

npm install -g react-native-cli

OK&But

通过以上这些工具的安装,我们手中的抹泥板就具备了堆砌React-Native-IOS应用的能力

但就像所有教程或指南一样,别人环境安装顺利不代表在你的开发环境上也一路绿灯

在此特别说明，必须安装xcode

  • 它会帮你安装必须的编译器
  • 总归你是开发ios应用,xcode还是能提供一些便利比如集成的手机模拟器

开发工具

通常xcode是御用的ios应用开发IDE但是我们可是要用React-Native啊,也就说我们要编写的基本是JS代码，xcode仍未对其编辑器添加比较好的JavaScript支持。以下是原书推荐和我所用的:

  • TextWrangler 这哥们用过 然后就没有然后了
  • WebStorm一直在用的 但可能不够cloud
  • Atom 据说很cloud 但……我一直不明白什么叫cloud

还有很多很多,相信高手总是能用记事本写出各种牛逼的程序……

为了不陷入开发工具之争,总之,选哪个都行,你开心就好^ ^!
