React-Native:快用我!

不同部分咋工作的呢

React-Native的卖点都有什么呢？

  1. 使用JavaScript编写IOS应用(目前应该支持安卓了,不信你看文件夹~)
  2. 所编写的应用原生运行、提供原生UI组件、原生的用户体验

ios系统本身并不能运行JavaScript，此前在ios系统上运行的原生应用必须用object-c来编写或者使用近来苹果推出的新语言-Swift,它也是要再进行编译才能在ios上运行.

能在ios运行JavaScript的只有两个东东:一个是Safari(屁话),另一个是UIWebView,这个UI组件可以放置到APP中，说白了就是套了个原生APP的壳子,里面是一个web运行环境,这种类型的APP也被叫做混合应用,hybrid,熟悉吧,这种应用确实是原生的,但其中原生代码也仅仅是为了启动UIWebView这个含有浏览器解析引擎的view容器而已，在这个组件里可以正常运行含有样式、js脚本的网页.

这种东东的存在带来一个好处,就是可以利用构建web的经验来编写非常接近原生的应用.不过尽管如此,虽然UIWebView本身是原生组件但它所承载的内容却不是——仅仅是个网页而已.这也就是为什么混合应用(hybrid)总是说它媲美原生,然而「媲美」只是相似而已,毕竟比不上完全用oc编写的真正原生.

话说回来,在HelloWorld这个例子中,我们是写了JavaScript代码,但是并没有创建网页文件,也没有使用UIWebView来运行我们的代码.相反我们的代码运行在JavaScript内核上,这个内核被植入进应用中.听起来是不是有点迷糊?来来来我们继续看代码.

用xcode打开HelloWorld/AppDelegate.m文件,代码清单如下:

#import "AppDelegate.h"

#import "RCTRootView.h"

@implementation AppDelegate

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions
{
  NSURL *jsCodeLocation;

  /**
   * Loading JavaScript code - uncomment the one you want.
   *
   * OPTION 1
   * Load from development server. Start the server from the repository root:
   *
   * $ npm start
   *
   * To run on device, change `localhost` to the IP address of your computer
   * (you can get this by typing `ifconfig` into the terminal and selecting the
   * `inet` value under `en0:`) and make sure your computer and iOS device are
   * on the same Wi-Fi network.
   */

  jsCodeLocation = [NSURL URLWithString:@"http://localhost:8081/index.ios.bundle?platform=ios&dev=true"];

  /**
   * OPTION 2
   * Load from pre-bundled file on disk. To re-generate the static bundle
   * from the root of your project directory, run
   *
   * $ react-native bundle --minify
   *
   * see http://facebook.github.io/react-native/docs/runningondevice.html
   */

//   jsCodeLocation = [[NSBundle mainBundle] URLForResource:@"main" withExtension:@"jsbundle"];

  RCTRootView *rootView = [[RCTRootView alloc] initWithBundleURL:jsCodeLocation
                                                      moduleName:@"HelloWorld"
                                               initialProperties:nil
                                                   launchOptions:launchOptions];

  self.window = [[UIWindow alloc] initWithFrame:[UIScreen mainScreen].bounds];
  UIViewController *rootViewController = [[UIViewController alloc] init];
  rootViewController.view = rootView;
  self.window.rootViewController = rootViewController;
  [self.window makeKeyAndVisible];
  return YES;
}

@end

AppDelegate.m文件是app的入口文件,就是在这个文件里到原生object-c代码和React-Native-JavaScript建立了联系.在第12行头文件引入了RCTRootView组件.

RCTRootView是React-Native框架提供的object-c编写的类,我们的js代码就是被它获取到并执行的.除此之外,它还能让我们用js代码来调用原生的iosUI组件.

这意味着控制应用的代码实际上都是JavaScript编写的，不会被转换为object-c、swift或者其他什么.呈现到我们设备上的界面也全部由UIKit的原生UI库提供,就像你使用传统object-c和swift一样.

这也解释了为什么我们能轻易的重载我们的程序.如果我们仅仅是更改了javascript代码,而没有修改object-c代码，那也就没有必要重新编译.RCTRootView能够监听command+r操作、对UI的更改做出反应、获取最近修改的JavaScript代码(如何完成这个动作我们后面说),最后执行新的代码.我们的应用也就很轻易的保持实时运行.
